Preguntas lab 02


  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    2. El uso de `!important` en CSS debería limitarse a situaciones donde sea absolutamente necesario anular estilos específicos y no se pueda lograr de otra manera. Su uso excesivo puede complicar la gestión y el mantenimiento del código CSS, ya que puede causar conflictos y dificultar la depuración de estilos. Es preferible utilizar otras alternativas para evitar depender de `!important.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    3. La selección cuidadosa de una imagen de fondo es importante por varias razones. Primero, la imagen debe estar optimizada para la web para asegurar tiempos de carga rápidos y un consumo de ancho de banda razonable. Además, la elección de la imagen debe tener en cuenta la resolución y el tamaño de la misma para que esta se pueda ver bien en diferentes dispositivos y pantallas. También es importante considerar el contenido y el contexto de la página para asegurarse de que la imagen de fondo sea relevante.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    4. La elección de unidades depende del contexto y del diseño específico de la página.

    • `%`: Es útil para crear diseños fluidos y responsivos, ya que se ajusta automáticamente al tamaño del contenedor o de la ventana del navegador
    • `px`: Proporciona un control preciso sobre el tamaño de los elementos, pero puede no adaptarse bien a diferentes tamaños de pantalla.
    • `pt`: Se usa principalmente en impresión y no es comúnmente utilizado en diseño web debido a su comportamiento fijo y poco adaptable a diferentes dispositivos.

  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    5. La versión minimizada del CSS mejora el rendimiento del sitio web al reducir el tamaño del archivo CSS descargado por los navegadores. Al eliminar espacios en blanco, comentarios y otros caracteres innecesarios, se reduce el tamaño del archivo, lo que resulta en tiempos de carga más rápidos. Esto hace una gran diferencia al cargar sitios web con una gran cantidad de estilos. Además, la minimización del CSS también puede ayudar a reducir el consumo de ancho de banda y los costos de alojamiento.
