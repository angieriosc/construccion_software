Preguntas lab 01


  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. El internet es una inmensa red global de computadoras interconectadas en todo el mundo. Se creo en 1960 (Guerra Fría) y funciona como una infrestructura que permite la comunicación entre dispositivos. Mientras que la World Wide Web es un sistema de información que funciona a través de internet. Se creo en 1989 por Tim Berners-Lee y permite transmitir diversos tipos de datos mediante el Protocolo de Transferencia de Hipertextos HTTP.

  2. ¿Cuáles son las partes de un URL?
    • Protocolo: Indica como se debe acceder al recurso (HTTP y HTTPS comunes). Ejemplo - https://www.ejemplo.com
    • Dominio: La dirección principal del sitio web. Ejemplo - www.ejemplo.com
    • Subdominio: Parte opcional que procede del dominio principal. Ejemplo - blog.ejemplo.com
    • Ruta: Especifica la ubicación del recurso dentro del sitio web. Ejemplo - /articulos/nuevo-articulo
    • Parámetros: Valores adicionales que se envían al servidor (opcional). Ejemplo - ?id=1238categoria=tecnologia
    • Fragmento: Se utiliza para enlazar una sección especifica de la página (opcional). Ejemplo - #seccion-interesante

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: Se utiliza para recuperar datos del servidor, solicita una representación de un recurso específico (No causa cambios en el estado del servidor)
    • HEAD: Es similar a Get pero solo solicita las cabeceras de la respuesta, sin recuperar el cuerpo del recurso
    • POST: Se utiliza para enviar datos a un recurso específico. Puede causar cambios en el estado del servidor (crea o autoriza un recurso) Enviar un formulario es un ejemplo.
    • PUT: El metodo Put remplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. Es muy bueno para actualizar completamente un recurso existente. Por, ejemplo al editar una entrada en el blog.
    • PATCH: Se utiliza para aplicar modificaciones parciales a un curso. Por ejemplo cambiar solo el título de la publicación.
    • DELETE: Barra un recurso específico. Un ejemplo es borrar un comentario en una publicación.

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. Se debe utilizar el método POST ya que te permite enviar datos desde un formulario HTML al servidor. A diferencia del método GET que coloca los datos en la URL, el método post los envia de manera más eficiente y segura en el cuerpo de la solicitud.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    6. El método GET ya que cuando escribes la dirección URL en tu navegador este se conecta con el servidor Web y envia una petición Get.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    7. Esto significa que la solicitud HTTP fue exitosa. El código 200 significa ok por lo que la solicitud del cliente fue recibida, entendida y aceptada correctamente por el servidor. No ocurrió ningún error.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    8. Sí ya que un error 404 según el protocolo HTTP Indica que el recurso solicitado no fue encontrado en el servidor punto esto puede ser por un enlace roto, una URL mal escrita o un recurso eliminado.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    9. Sí un error 500 indica un error interno del servidor lo que significa que algo salió mal en el servidor mientras procesaba la solicitud de usuario.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    10. Cuando un atributo HTML5 está depreciado significa que se considera obsoleto y se desaconse ja su uso futuro en favor de alternativas más modernas y estándares. Algunos elementos de HTML 4 que fueron desaprobados en HTML5 incluyen:

    • En HTML5, se desaconseja el uso del elemento `font` para aplicar estilos de texto. En su lugar, se recomienda utilizar CSS para estilizar el texto.
    • El elemento `center` se utilizaba para centrar contenido en HTML 4, pero en HTML5 se desaconseja su uso. En su lugar, se recomienda usar CSS para centrar elementos.
    • Los elementos relacionados con el uso de marcos (`frame`, `frameset`, `noframes`) están desaprobados en HTML5 debido a que el modelo de marcos ya no se considera una práctica recomendada en el desarrollo

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 introduce una serie de nuevos elementos semánticos como `header`, `footer`, `nav`, `article`, `section`, entre otros, que ayudan a estructurar y describir el contenido de una página web de manera más significativa.
    • HTML5 proporciona etiquetas específicas para la incorporación de audio y video nativo en las páginas web, como `audio` y `video`.
    • HTML5 incluye nuevos tipos de entrada de formularios, como `email`, `url`, `tel`, `date`, `number`, etc., así como validación de formularios del lado del cliente.
    • HTML5 ofrece nuevas APIs que permiten a los desarrolladores web crear aplicaciones más ricas y dinámicas, incluyendo la API de almacenamiento local, la API de geolocalización, la API de arrastrar y soltar, entre otras.

  11. ¿Qué componentes de estructura y estilo tiene una tabla?

    12. Las tablas en HTML pueden contener los siguientes componentes:

    • Etiqueta `table`: Define la tabla en sí misma.
    • Etiqueta `tr` (table row): Define una fila en la tabla.
    • Etiqueta `td` (table data): Define una celda de datos en una fila.
    • Etiqueta `th` (table header): Define una celda de encabezado en una fila o columna.
    • Etiqueta `thead`, `tbody`, `tfoot`: Estas etiquetas permiten organizar el contenido de la tabla en secciones: encabezado, cuerpo y pie de tabla.
    • Atributos y estilos CSS: Las tablas pueden ser estilizadas mediante atributos y reglas CSS para definir bordes, colores, alineación, espaciado, etc.

  12. ¿Cuáles son los principales controles de una forma HTML5?
    • `input type="text"`: Para entrada de texto.
    • `input type="password"`: Para contraseñas.
    • `input type="email"`: Para direcciones de correo electrónico.
    • `input type="number"`: Para números.
    • `input type="date"`, `input type="time"`, `input type="datetime-local": Para selección de fechas y horas.
    • `input type="checkbox"`: Para casillas de verificación.
    • `input type="radio"`: Para botones de opción.
    • `select` y `option`: Para listas desplegables.
    • `textarea`: Para áreas de texto multilínea.

  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/

  14. ¿Cuál es el ciclo de vida de los sistemas de información?

    15. El ciclo de vida de los sistemas de información es un proceso que describe las etapas necesarias para desarrollar, implementar y mantener sistemas de información. A continuación, te presento las siete fases clave:
    Planificación: Se definen los objetivos del sistema, se evalúan los recursos necesarios y se establece un cronograma. También se consideran aspectos como la seguridad, el presupuesto y la viabilidad del proyecto.
    Análisis de requerimientos: Se investigan y documentan las necesidades del sistema. Los analistas interactúan con los usuarios para comprender sus expectativas y definir los requisitos funcionales y no funcionales.
    Diseño: Se crea una estructura sólida para el sistema. Se diseñan la arquitectura, las interfaces de usuario y los componentes del sistema.
    Desarrollo: Aquí se implementa el diseño mediante programación. Los desarrolladores escriben código, crean bases de datos y construyen el software.
    Pruebas: Se verifica que el sistema funcione correctamente y cumpla con los requisitos. Las pruebas incluyen pruebas unitarias, de integración y de aceptación.
    Integración y ejecución: El sistema se pone en funcionamiento en el entorno real. Se realizan ajustes finales y se integra con otros sistemas si es necesario.
    Operación y mantenimiento: El sistema se utiliza en las operaciones diarias. Se realizan actualizaciones, correcciones de errores y mejoras según sea necesario.

  15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    16. El ciclo de desarrollo se centra en el proceso de construcción o creación de un producto, software o proyecto. Se refiere a las etapas específicas por las que pasa un proyecto desde su inicio hasta su finalización. En el desarrollo de software, esto implica fases como diseño, implementación, pruebas y lanzamiento. Es más específico y se relaciona con la metodología utilizada para llevar a cabo el proyecto
